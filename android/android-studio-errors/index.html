This is a warning saying that the ADB binary found is obsolete. It is not a critical error.

Action to take

  • Check the ADB version.

The ADB's location is found in the alert box..

Navigate to the platform-tools directory where adb.exe is located.

Execute the following command in the Command Prompt.

adb.exe --version

  • The adb version used is likely for an older SDK

Possible Solution

  • Uninstall older SDK build tools versions, leaving only the most recent one

Step 1: Open Android Studio and navigate to the Android SDK panel. From the top menu, choose Tools â†’ SDK Manager.

Step 2: In Android SDK main window, select SDK Tools tab.

Step 3: Check Show Package Details. You will see the build tools that you have.

Step 4: Uncheck the older built tools and click OK.

Step 5: Click OK to proceed with the deletion.

Step 6: Click Finish. You can run your app now.

If you get the following message, it means that VT-x is not enabled in your computer.

Action to take

  • Enable VT-x in your BIOS

Refer to the following video to see how to go about doing it.
